Template engine

Djangoâ€™s default template engine allows you to create dynamic web pages by separating the design from the data. This engine processes templates to render HTML dynamically based on context variables passed from the view.

1. Blocks: Blocks in Django templates are placeholders defined in a base template that child templates can override.

For example, in a base template:


{% block content %}
Content for the block.
{% endblock content %}

Child templates can then fill in the content for this block:


{% extends 'ex01/base.html' %}
{% block content %}
    <p>This is the content for the block.</p>
{% endblock content %}

2. Loops: The for template tag allows iteration over a sequence. For example:

To loop through a list of items:


{% for item in items %}
    <p>{{ item }}</p>
{% endfor %}

3. Conditional Statements: The if template tag allows for conditional rendering based on context variables.

For example, to display content based on a condition:


{% if user.is_authenticated %}
    <p>Welcome, {{ user.username }}!</p>
{% else %}
    <p>Please log in.</p>
{% endif %}

4. Context Variables: Context variables are passed from the view to the template to render dynamic content.

For example, in a Django view:


# In views.py
def my_view(request):
    context = {'message': 'Hello!'}
    return render(request, 'ex01/templates.html', context)

And in the template:


{% extends 'ex01/base.html' %}
{% block content %}
    <p>{{ message }}</p>
{% endblock content %}
