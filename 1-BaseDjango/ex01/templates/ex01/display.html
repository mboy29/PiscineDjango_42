Display process of a static page

When you request a static web page in Django, the process involves several steps, from the initial request to the final response. Here’s a step-by-step breakdown:

1. The Request: The process begins when a user makes a request to the server for a specific URL. For example, if a user navigates to http://example.com/display/, Django receives this request.

2. URL Routing: Django uses the URLconf to determine which view function should handle the request. The URLconf is a mapping between URL patterns and view functions defined in the Django project.

3. View Processing: The view function processes the request and returns an HTTP response. In the case of a static web page, the view function typically renders a template that contains the static content.

4. Template Rendering: The template engine processes the template and substitutes any template variables with their values. The resulting HTML content is sent back to the view function.

5. Response: The view function returns an HTTP response containing the rendered HTML content. This response is sent back to the user’s browser, which displays the static web page.

Here's an example of a view function in views.py:


from django.shortcuts import render

def display_view(request):
    return render(request, 'ex01/display.html')

And the URL routing in urls.py:


from django.urls import path
from . import views

urlpatterns = [
    path('display/', views.display_view, name='display'),
]
